# Passport Cover Engine

The permanent subsystem behind Worldly's Passport Covers: it generates the
cover artwork (switchable app icons), owns each cover's in-app theme (accent,
gradient, ambient particles), and validates that every layer of the system
stays in sync. A new collection should never require design direction beyond a
single prompt — everything else lives here.

## Anatomy

```
covers-engine/
  README.md            ← you are here
  DESIGN_SYSTEM.md     ← the design rules (composition, colour, motion…)
  PROMPTS.md           ← how to go from "Create a Nordic Collection" to shipped
  legacy/covers3.py    ← the original generator; provenance for the first 22
  src/
    geometry.ts        ← THE PROTECTED W — never edit (test-locked fingerprint)
    filters.ts         ← shared SVG filters + finish (vignette, gloss, grain)
    materials.ts       ← material library (porcelain, gold, ice, parchment…)
    mark.ts            ← renders the W in a material (geometry untouched)
    environments.ts    ← scene library (stars, skylines, ridges, stamps…)
    compose.ts         ← layered Scene → SVG compositor
    render.ts          ← Scene → PNG (resvg, no browser) + previews + sheets
    recipes/           ← one file per collection; each cover = a Scene function
    catalog.ts         ← every cover's THEME (accent/gradient/particles/androidBg)
    gen.ts             ← emits src/lib/coverThemes.gen.ts for the app runtime
    build.ts           ← renders engine-owned covers + review contact sheet
    check.ts           ← cross-checks catalog ⟷ app.json ⟷ assets on disk
```

In-app runtime (shipped over the air):

- `src/lib/coverThemes.gen.ts` — generated theme registry (do not hand-edit)
- `src/hooks/useCoverTheme.ts` — the active cover's theme, anywhere in the app
- `components/CoverParticles.tsx` — the animation engine (snow, stars, aurora…)
- `app/covers.tsx` — the Covers screen; header is themed + animated live

## Commands

```
npm run covers:build     # render every engine-owned cover + contact sheet
npm run covers:gen       # regenerate src/lib/coverThemes.gen.ts from catalog.ts
npm run covers:check     # validate catalog ⟷ app.json ⟷ assets
npm test                 # includes the geometry lock + theme integrity tests
```

Rendering is fully deterministic (seeded PRNG, resvg, no browser) — the same
recipe produces byte-identical art on any machine, forever.

## How a cover works

A cover is a **Scene** — a layered composition, never a flat drawing:

```
background → environment → the mark (in a material) → foreground → finish
```

The **mark** is the protected Worldly W (`geometry.ts`). Recipes may choose a
material (surface finish) and the five pin-hole colours; they may never alter
the geometry. The **finish** (vignette + gloss in SVG, photographic grain at
the raster stage) is applied to every cover so the whole collection feels like
one physical product line.

## Adding a collection (the short version)

1. Add a recipe file in `src/recipes/` (copy `voyager.ts` as a template).
2. Add each cover's theme to `catalog.ts`.
3. `npm run covers:build && npm run covers:gen` — review the contact sheet.
4. Register the icons in `app.json` (expo-alternate-app-icons plugin block)
   and add a section/covers to `src/lib/covers.ts` (with previews + unlock
   rules/season as needed).
5. `npm run covers:check && npm test` — both must pass.
6. Ship. Theme + catalog changes are OTA-safe immediately; the **icon assets
   themselves need the next native build** (the icon catalog is compiled into
   the binary). The Covers screen already degrades gracefully — tapping a
   too-new cover on an old binary explains it arrives with the next update.

The long version, including copy/tone rules and per-season guidance, is in
`PROMPTS.md`.

## Invariants (enforced)

- `geometry.ts` is fingerprint-locked by a unit test. Editing it fails CI.
- `covers:check` fails if any icon exists in app.json without a theme, any
  theme lacks an icon entry, or any asset (icon or preview) is missing.
- Every theme's colours are validated (hex, 2-stop gradient) by unit test.
- Reduce Motion: `CoverParticles` renders a static frame when the user has
  motion reduced — recipes never need to think about it.

## The first 22 covers

The pre-engine covers (Classic through Pride) were generated by
`legacy/covers3.py` (kept for provenance) and are user-approved shipped art —
they are catalogued and themed here but intentionally NOT re-rendered. New
work must go through the engine.
